🛠️ The Tech Stack

Application: FastAPI, Python 3.11, Uvicorn, pytest

Infrastructure: Minikube (Docker driver), K8s Deployments/Services, Nginx Ingress

Automation: Jenkins (Running as a Docker container on the host)

Version Control: Git

⚙️ Pipeline Requirements (The Jenkinsfile)
The Jenkinsfile must be written using Declarative Pipeline syntax and execute the following stages:

Checkout: Pull the latest code from the Git repository.

Test: Spin up a temporary Python environment or container to run pytest. If tests fail, abort the pipeline.

Build: Build the Docker image. Critical Constraint: Because this is a local Minikube cluster, the pipeline must either load the image into Minikube's internal Docker daemon (minikube image load) OR build it directly inside the Minikube environment so the K8s cluster can access it without a remote registry like Docker Hub.

Deploy: Update the Kubernetes deployment to use the newly built image. It must trigger a K8s Rolling Update.

Verify: Use kubectl rollout status to wait and verify that the 3 replicas successfully spun up and passed their /health readiness probes without dropping traffic.

🚀 Required Deliverables from the LLM

Jenkins Infrastructure Setup: Provide the exact docker run or docker-compose.yml command to spin up a Jenkins container on my Windows/WSL2 host. Crucial: This Jenkins container must have Docker CLI and kubectl installed, and it must have the correct volume mounts/permissions to communicate with my host's Docker daemon and Minikube cluster.

The Jenkinsfile: Provide the complete, production-grade Jenkinsfile fulfilling all the stages mentioned above.

Execution Steps: Provide a brief step-by-step guide on how to add this pipeline to the Jenkins UI once the server is running.
