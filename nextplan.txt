Stepping up to JMeter is exactly how enterprise engineers handle load testing. It is incredibly powerful because it gives you granular control over how the stress is applied to your server.

Since you are running on Windows and already have the JDK and JMeter installed, here is the exact step-by-step guide to configuring your test for 10,000 concurrent purchases against your Dockerized FastAPI app.

### ⚙️ Step 1: Launch JMeter

1. Navigate to your installed JMeter folder.
2. Go into the `bin` directory.
3. Double-click `jmeter.bat` to launch the graphical interface.

### 👥 Step 2: Create a Thread Group (The Virtual Users)

The "Thread Group" represents your users.

1. In the left panel, right-click on **Test Plan**.
2. Hover over **Add** -> **Threads (Users)** -> Click **Thread Group**.
3. In the new window, configure these exact settings:
* **Number of Threads (users):** `10000`
* **Ramp-up period (seconds):** `10`
* **Loop Count:** `1`



> **Why a 10-second ramp-up?** If you set this to 0, JMeter attempts to open 10,000 TCP ports on your Windows machine in a single millisecond. Windows will usually block this to protect itself (a phenomenon called Ephemeral Port Exhaustion), and the test will fail before it even reaches Docker. A 10-second ramp-up means JMeter will add 1,000 new users every second, which is a massive but realistic spike!

### 🌐 Step 3: Configure the HTTP Request

Now we tell those 10,000 users exactly where to click.

1. Right-click on your newly created **Thread Group**.
2. Hover over **Add** -> **Sampler** -> Click **HTTP Request**.
3. Fill out the web server details:
* **Protocol [http]:** `http`
* **Server Name or IP:** `127.0.0.1`
* **Port Number:** `8000`


4. Fill out the specific API endpoint:
* **HTTP Request Method:** `POST`
* **Path:** `/purchase/1`



### 📊 Step 4: Add Listeners (The Results)

Listeners capture the data so you can see if the database locked up or succeeded.

1. Right-click your **Thread Group**.
2. Hover over **Add** -> **Listener** -> Click **Summary Report**.
3. Right-click your **Thread Group** again.
4. Hover over **Add** -> **Listener** -> Click **View Results Tree**.

### 🚀 Step 5: Run the Stress Test

1. Make sure your Docker container is running in the background.
2. Click the green **Play Button** (Start) in the top toolbar of JMeter. (It may ask you to save the test plan first; you can just save it to your desktop).
3. Click on the **Summary Report** in the left panel to watch the numbers roll in live.

### ⚠️ A Critical Warning for JMeter Memory

The **View Results Tree** listener saves the entire payload of every single request. Storing 10,000 full HTTP responses in RAM can sometimes crash the JMeter application itself! If JMeter freezes during the test, delete the "View Results Tree" listener, leave only the "Summary Report", and run it again.
