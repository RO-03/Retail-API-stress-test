How to Improve It Further (Nginx Tuning)
While the architecture is solid, we can squeeze more performance out of Minikube by tuning how Nginx talks to your Uvicorn workers.

By default, Nginx closes the TCP connection to your API pods after every single request. Under a 10,000-request stress test, this means Nginx is constantly doing expensive "handshakes" with Uvicorn. We can fix this by enabling Upstream Keep-Alive Connections.

Open your k8s/ingress.yaml file and add these specific tuning annotations under the metadata section:

YAML
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: retail-api-ingress
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /
    # --- PERFORMANCE TUNING ANNOTATIONS ---
    # Keep 64 connections open to your Uvicorn pods to avoid TCP handshake overhead
    nginx.ingress.kubernetes.io/upstream-keepalive-connections: "64"
    nginx.ingress.kubernetes.io/upstream-keepalive-requests: "1000"
    nginx.ingress.kubernetes.io/upstream-keepalive-timeout: "60"
spec:
  rules:
    - host: retail-store.local
      http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: retail-api-service
                port:
                  number: 8000
To apply this change:

PowerShell
kubectl apply -f k8s/ingress.yaml
